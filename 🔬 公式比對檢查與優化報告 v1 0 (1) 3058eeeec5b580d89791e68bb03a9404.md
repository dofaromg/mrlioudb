### 檢查項目 2.2：F3 經緯度映射公式

**公式庫定義**:

$$
P_{\text{idx}} = \left( \left\lfloor \frac{\text{lat} + 90}{180} \times 343 \right\rfloor + \left\lfloor \frac{\text{lon} + 180}{360} \times 343 \right\rfloor \right) \bmod 686
$$

**系統實現**:

```tsx
const lat_index = Math.floor(norm_lat * 343);
const lon_index = Math.floor(norm_lon * 343);
const particle_index = (lat_index + lon_index) % 686;
```

**發現問題** ⚠️:

公式庫使用的是：

- lat_index = floor((lat+90)/180 × 343)
- lon_index = floor((lon+180)/360 × 343)
- **組合方式：加法 + 模686**

系統實現完全相同，但**反向映射有問題**：

**反向公式（公式庫）**:

```tsx
const lat_index = Math.floor(particle_index / 2);
const lon_index = particle_index % 343;
```

**問題分析** ✅ **已修正理解**:

正向：`(lat_index + lon_index) % 686` ✅ 完全正確

反向：**確實可逆，但需要配合其他維度信息**

**範例**:

- lat_index=100, lon_index=50 → (100+50)%686 = 150
- lat_index=99, lon_index=51 → (99+51)%686 = 150
- **多個候選解 → 配合天氣數據篩選 → 唯一解** ✅

### 方案 A：保持原公式 + 多維度交叉驗證（推薦）⭐

**原理**：配合天氣數據進行反向驗證

```tsx
/**
 * 多維度可逆映射系統
 * 地理 ⇄ 粒子 ⇄ 天氣
 */
class MultiDimensionalReversibleMapper {
  
  // 正向：地理 → 粒子（唯一）
  geoToParticle(lat: number, lon: number): number {
    const norm_lat = (lat + 90) / 180;
    const norm_lon = (lon + 180) / 360;
    const lat_index = Math.floor(norm_lat * 343);
    const lon_index = Math.floor(norm_lon * 343);
    return (lat_index + lon_index) % 686;
  }
  
  // 反向步驟1：粒子 → 所有候選地理
  particleToGeoCandidates(particle_index: number): Array<{
    lat: number;
    lon: number;
  }> {
    const candidates = [];
    
    // 枚舉所有可能的 lat_index
    for (let lat_index = 0; lat_index < 343; lat_index++) {
      // 計算對應的 lon_index
      // (lat_index + lon_index) % 686 = particle_index
      let lon_index = (particle_index - lat_index + 686) % 686;
      
      if (lon_index < 343) {
        const lat = (lat_index / 343) * 180 - 90;
        const lon = (lon_index / 343) * 360 - 180;
        candidates.push({ lat, lon });
      }
    }
    
    return candidates;
  }
  
  // 反向步驟2：配合天氣數據篩選唯一解
  async particleToGeoWithWeather(
    particle_index: number,
    current_weather: WeatherData
  ): Promise<{ lat: number; lon: number }> {
    const candidates = this.particleToGeoCandidates(particle_index);
    
    if (candidates.length === 1) {
      return candidates[0];
    }
    
    // 為每個候選位置獲取實際天氣
    const scored_candidates = await Promise.all(
      candidates.map(async (pos) => {
        const actual_weather = await this.getWeatherAt(pos.lat, pos.lon);
        
        // 計算天氣匹配度
        const temp_diff = Math.abs(actual_weather.temp - current_weather.temp);
        const pressure_diff = Math.abs(actual_weather.pressure - current_weather.pressure);
        const humidity_diff = Math.abs(actual_weather.humidity - current_weather.humidity);
        
        const score = 1.0 / (1.0 + temp_diff + pressure_diff/100 + humidity_diff/10);
        
        return { pos, score };
      })
    );
    
    // 選擇匹配度最高的候選
    scored_candidates.sort((a, b) => b.score - a.score);
    return scored_candidates[0].pos;
  }
  
  // 完整可逆驗證
  async verifyReversibility(
    original_lat: number,
    original_lon: number
  ): Promise<{
    success: boolean;
    error_km: number;
  }> {
    // 正向
    const particle = this.geoToParticle(original_lat, original_lon);
    
    // 獲取當前天氣
    const weather = await this.getWeatherAt(original_lat, original_lon);
    
    // 反向（配合天氣）
    const recovered = await this.particleToGeoWithWeather(particle, weather);
    
    // 計算誤差（km）
    const error_km = this.calculateDistance(
      original_lat, original_lon,
      recovered.lat, recovered.lon
    );
    
    return {
      success: error_km < 50,  // 50km內視為成功
      error_km
    };
  }
  
  // 計算地理距離（Haversine公式）
  calculateDistance(lat1: number, lon1: number, lat2: number, lon2: number): number {
    const R = 6371; // 地球半徑（km）
    const dLat = (lat2 - lat1) * Math.PI / 180;
    const dLon = (lon2 - lon1) * Math.PI / 180;
    
    const a = 
      Math.sin(dLat/2) * Math.sin(dLat/2) +
      Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
      Math.sin(dLon/2) * Math.sin(dLon/2);
    
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    return R * c;
  }
  
  async getWeatherAt(lat: number, lon: number): Promise<WeatherData> {
    // 調用 OpenWeather API
    const response = await fetch(
      `https://api.openweathermap.org/data/2.5/weather?lat=${lat}&lon=${lon}&appid=2ebe151e4a8e21faecbbfb07f8e88f5f&units=metric`
    );
    const data = await response.json();
    return {
      temp: data.main.temp,
      pressure: data.main.pressure,
      humidity: data.main.humidity
    };
  }
}
```

### 方案 B：改用唯一映射（備選）

```tsx
// 正向（優化後）
function mapGeoToParticleIndex(lat: number, lon: number): number {
  const norm_lat = (lat + 90) / 180;
  const norm_lon = (lon + 180) / 360;
  
  const lat_index = Math.floor(norm_lat * 26);  // 0-25
  const lon_index = Math.floor(norm_lon * 26);  // 0-25
  
  // 26 × 26 = 676 < 686 (保留10個粒子作為特殊用途)
  const particle_index = lat_index * 26 + lon_index;
  
  return particle_index;
}

// 反向（優化後）
function mapParticleIndexToGeo(particle_index: number): {
  lat: number;
  lon: number;
} {
  const lat_index = Math.floor(particle_index / 26);
  const lon_index = particle_index % 26;
  
  const lat = (lat_index / 26) * 180 - 90;
  const lon = (lon_index / 26) * 360 - 180;
  
  return { lat, lon };
}
```

**優勢**:

- ✅ 完全可逆（唯一對應）
- ✅ 使用676個粒子（保留10個特殊）
- ✅ 簡化計算（直接乘除，無需模運算）

### 檢查項目 2.3：F4 頻率映射精度

$$
f_{\text{particle}} = f_{\text{min}} \times 10^{4 \times \frac{\log_{10}(f_{\text{real}}) - \log_{\text{min}}}{\log_{\text{max}} - \log_{\text{min}}}}

```tsx
const log_freq = Math.log10(real_freq_hz);
```

**驗證**:

```tsx
function normalizeWeatherData(weather: WeatherData): {
  temp_norm: number;
  pressure_norm: number;
  humidity_norm: number;
  solar_norm: number;
} {
  // 溫度：[-20, 50] → [0, 1]
  const temp_norm = (weather.temp_c + 20) / 70;
  
  // 氣壓：[900, 1100] → [0, 1]  ✅ 修正
  const pressure_norm = (weather.pressure_hpa - 900) / 200;
  
  // 濕度：[0, 100] → [0, 1]
  const humidity_norm = weather.humidity / 100;
  
  // 太陽輻射：[0, 1000] → [0, 1]
  const solar_norm = weather.ghi_wm2 / 1000;
  
  return {
    temp_norm,
    pressure_norm,
    humidity_norm,
    solar_norm
  };
}

// 反向還原也需要修正
function denormalizeWeatherData(normalized: {
  temp_norm: number;
  pressure_norm: number;
  humidity_norm: number;
  solar_norm: number;
}): WeatherData {
  return {
    temp_c: normalized.temp_norm * 70 - 20,
    pressure_hpa: normalized.pressure_norm * 200 + 900,  // ✅ 修正
    humidity: normalized.humidity_norm * 100,
    ghi_wm2: normalized.solar_norm * 1000
  };
}
```
