## ⚠️ 已知問題與風險 / Known Issues & Risks

### 高優先級問題 🔴

#### 1. 安全問題：預設密碼

**問題描述：**
- MongoDB 密碼設為 `changeme123`
- 存儲在 Git 倉庫中的明文密碼

**風險等級：** 🔴 **高風險**

**影響：**
- 數據庫可能被未授權訪問
- 數據洩漏風險

**解決方案：**
```bash
# 立即執行
NEW_PASSWORD=$(openssl rand -base64 32)
kubectl create secret generic mongodb-secret \
  --from-literal=password=$NEW_PASSWORD \
  --namespace=flowagent \
  --dry-run=client -o yaml | kubectl apply -f -

# 保存密碼到安全位置
echo "$NEW_PASSWORD" | gcloud secrets create mongodb-password --data-file=-
```

**預計修復時間：** 5 分鐘

#### 2. 單點故障：MongoDB

**問題描述：**
- MongoDB 只有 1 個副本
- 無自動備份

**風險等級：** 🔴 **高風險**

**影響：**
- Pod 故障將導致服務中斷
- 數據丟失風險

**解決方案：**
```yaml
# 升級為 StatefulSet with ReplicaSet
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mongodb
spec:
  replicas: 3
  serviceName: mongodb
```

**預計修復時間：** 2-3 小時

### 中優先級問題 ⚠️

#### 3. 容器映像標籤

**問題描述：**
- 使用 `latest` 標籤而非語義化版本

**風險等級：** ⚠️ **中風險**

**影響：**
- 部署結果不可預測
- 難以回滾

**建議改進：**
```yaml
# 使用明確版本
image: asia-east1-docker.pkg.dev/flowmemorysync/flowagent/module-a:v1.2.3
```

#### 4. 缺少健康檢查

**問題描述：**
- 大部分 Deployment 缺少 liveness/readiness probes

**風險等級：** ⚠️ **中風險**

**影響：**
- 無法自動重啟失敗的容器
- 流量可能路由到未就緒的 Pod

**建議改進：**
```yaml
livenessProbe:
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10

readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
```

### 低優先級問題 💚

#### 5. 監控告警

**問題描述：**
- Prometheus 已部署但未配置告警規則

**建議改進：**
- 添加 Alertmanager
- 配置關鍵指標告警

#### 6. 日誌聚合

**問題描述：**
- 依賴 kubectl logs
- 無集中式日誌管理

**建議改進：**
- 啟用 Cloud Logging
- 或部署 Loki
