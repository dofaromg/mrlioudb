# Product Naming Convention
# 產品命名慣例

**MrLiou FlowCore ParticleRuntime** — v1.0.0 — MrLiouWord

---

## Purpose / 目的

This document describes how product identity is embedded directly in code
across the FlowCore runtime, API responses, CLI banners, trace metadata, and
artifact index outputs.

本文件說明產品身份識別如何直接嵌入 FlowCore 執行環境、API 回應、CLI 標語、
追蹤元資料及製品索引輸出中的方式。

The convention treats naming as a **first-class, product-level concern** —
the way mainstream products (Apple, Stripe, Vercel) embed brand/version
identity in their SDKs and runtimes rather than scattering ad-hoc strings
throughout the codebase.

此慣例將命名視為**一等的產品層級關切** — 如同主流產品（Apple、Stripe、Vercel）
將品牌與版本身份嵌入 SDK 與執行環境，而非散落於程式碼各處的臨時字串。

---

## Single Source of Truth / 單一真實來源

All naming values live in **`flowcore_naming.py`** at the repository root.

所有命名值集中於倉庫根目錄的 **`flowcore_naming.py`**。

```
flowcore_naming.py          ← single source of truth / 單一真實來源
├── PRODUCT dict            ← all identity constants / 所有身份常數
├── event_name()            ← namespaced event labels / 命名空間事件標籤
├── server_version_header() ← HTTP Server header / HTTP 伺服器標頭
├── server_banner()         ← startup console banner / 啟動主控台標語
├── index_metadata()        ← artifact manifest block / 製品清單區塊
├── health_metadata()       ← health-check payload block / 健康檢查有效載荷區塊
└── cli_description()       ← argparse help description / argparse 說明描述
```

## Where Product Identity Appears / 產品身份出現的位置

### 1. HTTP `Server` Header

The `RuntimeHandler` in `tools/ai-computer/ai_computer.py` sets:

```
Server: FlowCore.Runtime/1.0.0
```

This is generated by `server_version_header("runtime")`.

### 2. Startup Console Banner / 啟動主控台標語

When any FlowCore server process starts it prints:

```
MrLiou FlowCore ParticleRuntime [FlowCore.Runtime] v1.0.0 — MrLiouWord
Listening on http://127.0.0.1:8765
```

Generated by `server_banner("runtime")`.

### 3. CLI `--help` Description

```
$ python tools/ai-computer/ai_computer.py --help
usage: ai_computer.py ...

MrLiou FlowCore ParticleRuntime [FlowCore.Runtime] v1.0.0 — Particle Language Core Runtime / 粒子語言核心系統
```

Generated by `cli_description("runtime")`.

### 4. Health Endpoint / 健康檢查端點

`GET /health` (ai_computer) and `GET /health` (flowcore_loop) include:

```json
{
  "status": "ok",
  "product": "MrLiou FlowCore ParticleRuntime",
  "component": "FlowCore.Runtime",
  "version": "1.0.0",
  "origin_signature": "MrLiouWord"
}
```

`GET /judge/health` (MrLiou_AI_SuperComputer) includes the same block.

Generated by `health_metadata("runtime" | "ai" | "web")`.

### 5. Artifact Index / 製品索引

`POST /index` (ai_computer) writes `artifacts/index/manifest.json` with a
top-level `product` block:

```json
{
  "generated_at": "...",
  "root": "/path/to/vault",
  "file_count": 42,
  "merkle_root": "...",
  "product": {
    "product": "MrLiou FlowCore ParticleRuntime",
    "vendor": "MrLiou",
    "namespace": "mrliou.flowcore",
    "version": "1.0.0",
    "origin_signature": "MrLiouWord",
    "slug": "mrliou-flowcore"
  },
  "files": [...]
}
```

Generated by `index_metadata()`.

### 6. Trace Event Namespace / 追蹤事件命名空間

Every trace record in `artifacts/trace/trace.jsonl`
(ai_computer) gains an `event` field alongside the existing `operation`:

```jsonl
{"tick": 1, "operation": "health", "event": "mrliou.flowcore.trace.health", ...}
```

Every trace record in `log/trace.jsonl`
(MrLiou_AI_SuperComputer) gains an `ns_event` field alongside the existing `event`:

```jsonl
{"tick": 1, "event": "judge_health", "ns_event": "mrliou.flowcore.judge.health", ...}
```

Backward compatibility is preserved: existing consumers of the short event
name field continue to work unchanged.

## How to Update the Version / 如何更新版本

1. Open `flowcore_naming.py`.
2. Change the `"version"` key in the `PRODUCT` dict.
3. All banner strings, HTTP headers, health responses, and index manifests
   automatically reflect the new version.

---

## How to Add a New Component / 如何新增元件

1. Add a key/value pair to `PRODUCT["component"]` in `flowcore_naming.py`.
2. Use `server_banner("my_component")`, `health_metadata("my_component")`, etc.
3. No other files need to be updated for naming consistency.

---

## Backward Compatibility / 向後相容性

- All changes are **additive** — no existing field is renamed or removed.
- Existing short event names in traces are preserved; the new `ns_event` /
  `event` (namespaced) field is added alongside them.
- Any service or tool not aware of `flowcore_naming` falls back to the
  inline defaults defined at the top of each file.
