## 🔒 安全性評估 / Security Assessment

### 安全等級：⚠️ 中等 (需改進)

#### 高風險項目 🔴

1. **明文密碼存儲**
   - 問題：MongoDB 密碼硬編碼在 YAML
   - 風險：GitHub 公開倉庫可能洩漏密碼
   - 解決方案：使用 Google Secret Manager

2. **預設密碼**
   - 問題：`password: changeme123`
   - 風險：極易被暴力破解
   - 解決方案：立即更換為強密碼

3. **缺少 Network Policy**
   - 問題：Pod 間無限制通信
   - 風險：橫向移動攻擊
   - 解決方案：實施 NetworkPolicy

#### 中風險項目 ⚠️

1. **使用 'latest' 標籤**
   - 問題：不可預測的部署結果
   - 解決方案：使用語義化版本標籤

2. **無 Pod Security Policy**
   - 問題：缺少 Pod 安全限制
   - 解決方案：啟用 Pod Security Standards

3. **無映像掃描**
   - 問題：可能包含已知漏洞
   - 解決方案：啟用 Container Scanning

#### 低風險項目 💚

1. **LoadBalancer 暴露**
   - 現狀：HTTP 而非 HTTPS
   - 建議：配置 SSL/TLS 證書

### 安全改進優先級

**第一優先 (立即執行)：**
1. 更換所有預設密碼
2. 遷移 Secrets 到 Google Secret Manager
3. 實施 NetworkPolicy

**第二優先 (1週內)：**
1. 配置 SSL/TLS 證書
2. 啟用 Pod Security Standards
3. 實施 RBAC 細粒度權限

**第三優先 (1個月內)：**
1. 映像漏洞掃描
2. Binary Authorization
3. Audit Logging

```
基礎成本 = (節點數 × 節點成本 × 730小時) + GKE管理費 + 存儲費用
網路成本 = (出站流量GB × $0.12) + (LoadBalancer數 × $20)
```

### C. 故障排除速查表

| 問題 | 診斷命令 | 常見原因 | 解決方案 |
|------|---------|---------|---------|
| Pod 無法啟動 | `kubectl describe pod <pod>` | 映像拉取失敗 | 檢查 Registry 權限 |
| Service 無法訪問 | `kubectl get svc -o wide` | LoadBalancer pending | 等待 5-10 分鐘 |
| 高記憶體使用 | `kubectl top pods` | 記憶體洩漏 | 重啟 Pod 或調查應用 |
| 慢響應時間 | `kubectl logs -f <pod>` | 數據庫連接慢 | 優化查詢或加快取 |
| 部署失敗 | `kubectl get events` | 配置錯誤 | 檢查 YAML 語法 |
